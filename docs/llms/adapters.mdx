## Built-in Providers

- `openai`
- `litellm`
- `anthropic_agent`

## Provider Registry

```python
from afk.llms import list_llm_providers, get_llm_provider

print(list_llm_providers())
provider = get_llm_provider("openai")
```

## Creating Clients

```python
from afk.llms import create_llm_client

llm = create_llm_client(provider="litellm")
```

## Custom Provider

Implement `LLMProvider` and register:

```python
from afk.llms import register_llm_provider

register_llm_provider(my_provider)
```

## Provider Implementation Contract

Your provider should:

- validate provider-specific settings via `ProviderSettingsSchema`
- build one transport implementing chat/stream/embed/session methods
- only expose normalized AFK request/response types
- raise explicit capability errors for unsupported features

`afk.llms.clients` is the adapter layer:

- `clients/contracts.py` defines transport protocol boundaries
- `clients/base/responses.py` provides shared Responses-style adapter logic
- `clients/shared/*` contains reusable content/normalization/transport helpers

This separation keeps adapter behavior modular:

- provider-specific transport code stays in `clients/adapters/*`
- shared wire-format conversion lives in `clients/shared/*`
- runtime policies stay in `llms/runtime/*` (not inside adapters)
